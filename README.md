# PatternsLab

Интерактивная платформа для изучения **паттернов проектирования** и архитектурных
подходов. Каждый паттерн реализован как независимый **микрофронтенд** со своим UI,
логикой и живым демо; модули подгружаются в host-оболочку в рантайме через
**Module Federation**.

## Цели

1. Показать паттерны проектирования через реальные frontend-сценарии.
2. Продемонстрировать микрофронтенд-архитектуру с независимыми сборкой и запуском модулей.
3. Сделать платформу легко расширяемой новыми паттернами.

## Технологии

- **React 19** + **TypeScript**
- **Vite** — сборка и dev-сервер
- **Module Federation** (`@module-federation/vite`) — загрузка микрофронтендов в рантайме
- **React Router** — навигация в host-оболочке
- **TailwindCSS** — стили
- **Framer Motion** — пошаговые анимации в визуализациях алгоритмов
- **prism-react-renderer** — подсветка синтаксиса в просмотрщике кода
- **pnpm workspaces** + **Turborepo** — монорепозиторий

## Архитектура

```
┌─────────────────────────────┐
│           host              │  оболочка: навигация, каталог, роутинг
│  (apps/host, :3000)         │  читает реестр и подгружает паттерны
└──────────────┬──────────────┘
               │ Module Federation (loadRemote)
   ┌───────────┴───────────────┐
   ▼                           ▼
┌─────────────────┐     ┌─────────────────┐
│  behavioral     │ ... │  creational     │   remote-микрофронтенды
│  (:3001)        │     │  (:300x)        │   по одному на категорию
│  Observer, ...  │     │  Singleton, ... │
└─────────────────┘     └─────────────────┘
```

- **host** не знает о паттернах напрямую — он читает **реестр** (`packages/core`) и по
  его данным строит меню и подгружает нужный модуль из remote-а.
- Каждый **remote** — самостоятельное приложение: его можно запустить и открыть отдельно
  (автономный режим), и он же отдаёт свои паттерны наружу для host.
- Гранулярность: **один remote на категорию** паттернов
  (порождающие / структурные / поведенческие / архитектурные / алгоритмические). Внутри каждый паттерн —
  изолированный модуль с единым контрактом `PatternModule`
  (`meta` + `Demo` + `Explanation` + `code`).
- На странице паттерна host показывает три **вкладки**: «Разбор» (теория), «Демо»
  (живой виджет) и «Код» (исходники реализации). Код подтягивается из реальных файлов
  паттерна через Vite-импорт `?raw`, поэтому он всегда совпадает с тем, что крутится в демо.

## Структура репозитория

```
PatternsLab/
├── apps/
│   ├── host/                     # host-оболочка (:3000)
│   └── patterns/
│       ├── behavioral/           # remote с поведенческими паттернами (:3001)
│       │   └── src/patterns/observer/   # Observer: логика + Demo + Explanation
│       └── algorithmic/          # remote с алгоритмическими приёмами (:3002)
│           └── src/patterns/two-pointers/  # Два указателя: трейсер + визуализация
├── packages/
│   └── core/                     # контракты (PatternMeta, PatternModule) и реестр
├── pnpm-workspace.yaml
├── turbo.json
└── tsconfig.base.json
```

## Запуск

Требуется **Node.js ≥ 20** и **pnpm**.

```bash
# установить pnpm (если ещё нет)
npm install -g pnpm

# установить зависимости
pnpm install

# запустить всё (host + все remote) через Turborepo
pnpm dev
```

После запуска:

- host — http://localhost:3000
- remote `behavioral` (автономный режим) — http://localhost:3001
- remote `algorithmic` (автономный режим) — http://localhost:3002

Для корректной загрузки паттернов в host соответствующий remote должен быть запущен.

Запуск отдельного приложения:

```bash
pnpm --filter @patterns-lab/host dev
pnpm --filter @patterns-lab/behavioral dev
pnpm --filter @patterns-lab/algorithmic dev
```

## Как добавить новый паттерн

1. Создать модуль в нужной категории, напр. `apps/patterns/behavioral/src/patterns/strategy/`
   с файлами логики, `Demo.tsx`, `Explanation.tsx` и `index.ts` (экспорт `PatternModule`).
   В `index.ts` собрать поле `code`, импортируя нужные файлы как строки (`import src from "./strategy.ts?raw"`).
2. Добавить экспорт в `exposes` в `vite.config.ts` remote-а.
3. Дописать запись в реестр `packages/core/src/registry.ts`.

Host подхватит паттерн автоматически — отдельный код в оболочке не нужен.
